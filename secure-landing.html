What It Does

The Inner I Secure Gateway is the governance layer for AI agents. It answers one question before every tool call: should this agent be allowed to do this?

🔒 Tool Policy Engine

OPA-inspired policy evaluation. Every tool call is checked against configurable rules. Fail-closed: if no rule matches, deny. Supports rate limiting, required params, and agent allowlists.

🛡️ Injection Detection

Scans agent outputs and tool calls for prompt injection, data exfiltration, and jailbreak patterns. 20+ detection patterns across 4 severity levels. Returns risk score 0-100.

📋 Audit Chain

Append-only audit log with HMAC-SHA256 integrity verification. Every policy evaluation and security scan is recorded. Tamper-evident. Verifiable.

API Endpoints

The Secure Gateway exposes a REST API for agent governance:

  • POST /v1/evaluate — Policy verdict (allow/deny + reason)
  • POST /v1/scan — Injection scan on text output
  • POST /v1/scan/tools — Injection scan on tool call log
  • GET /v1/audit — Audit chain entries
  • GET /v1/audit/verify — HMAC integrity verification
  • GET /v1/policy — List current policy rules
  • POST /v1/policy/rule — Add a policy rule at runtime
  • GET /health — Service health check
